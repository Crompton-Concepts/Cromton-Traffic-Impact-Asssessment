Cromton Traffic Impact Assessment (TIA)

Welcome to the comprehensive user manual for the Cromton TIA application. This browser-based traffic engineering tool is designed to help you assess work-zone and detour impacts, calculate queue lengths and Volume-to-Capacity Ratios (VCR) / Level of Service (LOS), and generate professional reports.

Launch Standard Calculator Launch Formula View Launch Developer View
Which view should I use?
• Standard (index.html): The primary interface for daily traffic assessment and report generation.
• Formula View (index_formulas.html): Best for auditing. It keeps all calculation steps and formulas visible throughout the workflow and in generated reports.
• Developer View (index_developer.html): Includes experimental beta features and isolated test surfaces.

1. Account & Data Synchronization

The application supports a seamless multi-device experience using secure account synchronization.

  • Sign In: When you open the application, you will be prompted to log in. This links your session to your account.
  • Cloud Sync: User data and preferences are synced via a secure backend (Firebase Realtime Database). This means you can start an analysis on one device and continue on another.
  • Offline Resilience: If you lose internet connection or the sync service is unavailable, the application gracefully falls back to local browser storage. Your work is saved locally and will sync when the connection is restored.

2. Loading Traffic Datasets

The TIA application relies on comprehensive regional traffic data to power its calculations. It automatically manages fetching and updating these datasets.

Supported Regions

Data is loaded dynamically for multiple regions across Queensland and New South Wales, including:

  • TMR (Department of Transport and Main Roads)
  • Brisbane, Gold Coast, Ipswich, Logan, Toowoomba, Tewantin
  • NSW Regional Data

Data Loading Process

  1. Upon loading the app, check the Sites Loaded badge at the top of the interface.
  2. The system fetches the latest data versions using a secure manifest (dataset_manifest.json). It ensures data integrity by verifying secure hashes (SHA-256).
  3. If live GitHub source data is unavailable, the app automatically falls back to local cached files to ensure you can keep working.
If the Sites Loaded badge shows 0 or an unexpectedly low number, press Ctrl+F5 to force a hard refresh and clear stale browser caches.

3. Core Analysis Workflow

Follow these steps to conduct a standard traffic impact assessment:

Step 3.1: Site Selection

  • Use the interactive map or the site search field to locate your work zone.
  • Click on a map marker or select a search result to load the site's historical traffic data.
  • Review the site metadata (Road Name, Directions, Counter Details) to ensure you have the correct location.

Step 3.2: Setting Parameters

Input your project-specific parameters into the analysis workspace:

  • Timeline: Set the Base Year (of the data) and the Target Year (for your analysis).
  • Growth: Apply expected annual growth rates.
  • Geometry: Configure lane settings, including total lanes and lanes closed during the work phase.
  • Assumptions: Adjust heavy vehicle percentages and directional traffic splits (Direction 1 vs Direction 2) based on local knowledge.

Step 3.3: Running Calculations

Once inputs are set, the browser-first calculation engine instantly processes the data.

  • Queue Estimates: View hourly queue lengths, incorporating standard methodologies (AGTTM and SWT variants).
  • Capacity Analysis: Review directional and hourly Volume-to-Capacity Ratios (VCR) and corresponding Level of Service (LOS).
  • Visualizations: Check the generated Chart.js graphs and data tables for clear trend analysis.

4. Detour Route Analysis

For projects requiring road closures, the application includes a Detour Route capacity modeling tool.

  • Use the map interface to map out proposed detour routes.
  • The system calculates the additional stress and capacity impacts on the selected detour roads by overlaying the diverted traffic volumes.
  • Review the detour capacity charts to ensure the alternative route can handle the expected load without severe congestion.

5. Report Generation & Export

The application offers two distinct pathways for generating professional reports, depending on your deployment setup.

Method A: Standard Frontend Export (Always Available)

  1. Switch the UI to Report Mode.
  2. The app will format your inputs, site details, tables, and charts into a printable layout.
  3. Use the browser's Print Preview (Ctrl+P) to review the layout.
  4. Select "Save as PDF" to export the final document.
If you used index_formulas.html, all underlying math and step traces will be explicitly printed in this report.

Method B: Advanced Python Backend Service (If Configured)

For more advanced narrative generation, the app connects to a local Python FastAPI service.

  • The frontend sends a structured payload containing all calculation results to the backend.
  • The backend generates an editable draft report, translating raw data into professional narrative text.
  • If configured with an API key, the service utilizes AI (Gemini) to enrich the executive summaries and narrative flow.
  • You can edit the text directly in the provided draft editor before final export.

6. Quality Assurance & Formula Verification

Accuracy is critical. The application includes built-in tools to verify calculations:

  • Formula Agent (formula-agent.js): A background test harness that validates live calculations against reference engineering formulas.
  • Live Auditing: By using the Formula View, engineers can manually audit the exact formulas applied to their specific site data, ensuring transparency in every assessment.

7. Troubleshooting & FAQ

Q: The map isn't loading or is completely blank.

A: Ensure you have an active internet connection, as the map relies on external Leaflet libraries. Check if any strict ad-blockers or browser extensions are preventing scripts from running.

Q: Why does the top badge say "0 Sites Loaded"?

A: This usually means the dataset cache is stale or the download failed. Perform a hard refresh (Ctrl+F5 or Cmd+Shift+R). If it persists, ensure you can access GitHub, as the app fetches raw data from there before trying local fallbacks.

Q: My calculation results look incorrect or highly skewed.

A: Double-check your inputs. Common errors include entering a Target Year earlier than the Base Year, inputting extreme growth rates, or incorrect Direction 1 / Direction 2 traffic percentage splits (they should sum to 100%).

Q: I need to show my calculation methodology to a reviewer. How do I do that?

A: Do not use the standard index.html. Instead, open index_formulas.html, re-run your analysis, and generate the report. This version explicitly renders the math steps in the final output.

Q: The advanced text report editor isn't working.

A: The advanced editable report requires the Python backend service (report_service.py) to be running on your machine or network. If it is not running, you must use the standard Frontend Export method (Print to PDF).
